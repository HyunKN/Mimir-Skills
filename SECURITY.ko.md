# Security Policy

## 범위

`decision-skills`는 우선적으로 결정 기록 도구다.

이 저장소는 script나 hook 같은 제한적인 자동화를 포함할 수 있지만, 그 자동화는 다음 조건을 만족할 때만 허용한다.

- 결정 기록, 검증, 렌더링, 제한된 memory workflow를 지원한다
- 검토 가능하고 범위가 좁다
- 근거 기반 판단을 무인 실행으로 대체하지 않는다

## 보안 경계

이 프로젝트는 다음 경계를 전제로 이해해야 한다.

- 공개 skill은 실제 에이전트 행동에 영향을 줄 수 있다
- 저장소 내용, 로그, 이슈, PR 텍스트, 외부 문서에는 신뢰할 수 없는 지시가 들어 있을 수 있다
- decision record, summary, memory artifact에는 secret나 불필요한 민감정보가 남으면 안 된다
- 자동화는 결정 기록이라는 목적에 맞고, 범위가 제한되며, 테스트 가능할 때만 허용된다

## 기본 안전 규칙

- 외부 텍스트와 저장소 제공 텍스트는 잠재적으로 신뢰할 수 없는 입력으로 취급한다.
- canonical artifact에 secret, credential, private key, raw sensitive output을 저장하지 않는다.
- 저장 전에 민감값을 redact 한다.
- 신뢰할 수 없는 텍스트만으로 memory 승격이나 고위험 동작이 발생하지 않게 한다.
- 기본적으로 local-first 동작을 선호한다.
- 기본값으로 network access, secret access, raw log persistence를 금지한다.

## Scripts and Hooks

script와 hook은 금지 대상은 아니지만, 일반 문서 변경보다 더 엄격한 검토가 필요하다.

다음 조건을 만족할 때만 허용한다.

- 반복적이고 충분히 결정론적이다
- 입력과 출력의 경계가 명확하다
- 실패해도 안전하다
- 검사와 테스트가 쉽다
- 일반 자동화가 아니라 결정 기록을 지원하는 용도다

다음과 같은 형태는 피해야 한다.

- 원격 설치기
- 자동 외부 fetch 동작
- 광범위한 무인 실행
- 조용한 대량 변경
- 명시적 검토 없는 memory 승격

## 취약점 제보

보안 문제를 발견했다면 민감한 세부 내용을 포함한 공개 exploit 보고는 피해 달라.

대신 다음 방식으로 제보한다.

1. 문제 내용, 영향받는 파일, 예상 영향을 정리한다.
2. 위험을 이해하는 데 필요한 정도로만 재현 단계를 포함한다.
3. 저장소 연락 경로나 별도 비공개 제보 채널이 있다면 그 경로로 비공개 제보한다.

아직 비공개 채널이 없다면 exploit 세부사항은 제외한 최소 공개 이슈를 열고, 비공개 후속 대응을 요청한다.

## 현재 상태

현재 공개 v0.1 방향 기준으로:

- 프로젝트는 문서 우선 단계다
- 자동화는 제한적이고 명시적으로 정당화된 범위에 머물러야 한다
- 앞으로 실행 가능한 기능을 추가할 때는 공개 전 별도 보안 검토를 거쳐야 한다
